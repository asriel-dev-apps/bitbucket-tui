# M3 実装仕様書 — リポジトリブラウズ

対象: `bitbucket-tui/`。上位方針は `../design.md`、規約は `../../CLAUDE.md`、環境事実・M0〜M2実装メモは `../LEDGER.md` を必ず読むこと。M0〜M2 の上に載せる。

## ゴール

選択リポジトリの中身を閲覧できるようにする: **ブランチ一覧 → コミット履歴 → コミット詳細/差分**、および **ソースツリー閲覧(ディレクトリ移動 + ファイル内容表示)**。M1 の Diff ビューア、M2 のログ/ページャを流用し、レビュー主目的の TUI にコード探索を足す。

## スコープ(M3でやること)

1. API クライアント拡張(refs/branches・commits・commit 詳細・commit diff・source[dir/file])。
2. 画面追加: `Branches`(一覧) / `Commits`(履歴) → `CommitDetail`(メタ) → `Diff`(commit 差分・M1流用) / `Source`(ツリー閲覧) → `FileView`(ファイル内容・ページャ流用)。
3. ナビゲーション: repo コンテキストから Branches / Source に入れるようにする。

### やらないこと(後回し)

- ファイル/ブランチの作成・編集・削除(閲覧専用)。tags/branching model の詳細。blame。検索。
- シンタックスハイライト(ファイル内容はプレーン表示。diff は M1 の +/- 着色を流用)。
- 大容量ファイル/バイナリの本格対応(バイナリは「(バイナリ表示不可)」、巨大ファイルは先頭 N 行 + 打切り表示で可)。

## 依存(重要)

**新規クレート追加は禁止**(依存凍結・ネットワーク不可)。既存依存のみ。ファイル内容表示は M2 の `logview`/M1 の diff ページャを流用。新クレートが要ると判断したら追加せず報告して止まること。

## API 仕様(追加)

base = `https://api.bitbucket.org/2.0`。`{ws}`/`{repo}` は slug。認証・ページング・エラー変換・生テキスト取得は M0〜M2 の `api` を再利用。既定ブランチは repo オブジェクトの `mainbranch.name`(M0 で取得済みなら流用、無ければ repo 詳細を取得)。

- ブランチ一覧: `GET /repositories/{ws}/{repo}/refs/branches?pagelen=50&sort=-target.date`
  - branch オブジェクト: `name`, `target`{hash, date, message, author}。
- コミット履歴: `GET /repositories/{ws}/{repo}/commits/{revision}?pagelen=50`(revision=ブランチ名/ハッシュ。省略時 `/commits` は既定ブランチ)。
  - commit: `hash`, `message`, `date`, `author`{raw, user?}, `parents`[{hash}]。
- コミット詳細: `GET /repositories/{ws}/{repo}/commit/{hash}`(単数 `commit`)。
- コミット差分(テキスト): `GET /repositories/{ws}/{repo}/diff/{spec}` → `text/plain` ユニファイド diff(`{spec}` は hash 単体で当該コミット差分。M1 の diff パーサ/着色を流用)。
- ソース(ディレクトリ/ファイル): `GET /repositories/{ws}/{repo}/src/{commit}/{path}`
  - **path がディレクトリ** → ページングされたエントリ一覧(`type`=`commit_directory`|`commit_file`, `path`, `size`, `mimetype?`)。
  - **path がファイル** → **生の内容**(バイト列)。`?format=meta` でメタのみ。
  - ルートは `path` 空。`{commit}` はブランチ名/ハッシュ。ディレクトリ判定は `type` で行う。

> **モデル(serde)**: `Branch`{name, target{hash, date?, message?, author?}}、`Commit`{hash, message?, date?, author{raw?, user?}, parents[]{hash}}、`SrcEntry`{type("commit_file"/"commit_directory"), path, size?, mimetype?}。id/hash 以外は `Option`/`#[serde(default)]`。バイナリ判定は mimetype や内容の NUL 有無で。フィールドは実 API で要検証 → LEDGER に反映。

必要スコープ: `repository`(読取。M0 の repo 一覧と同じ)。

## TUI 仕様(追加)

- `Screen` に `Branches`, `Commits`, `CommitDetail`, `Source`, `FileView` を追加(`Diff` は M1 のものを commit 差分にも再利用)。
- **ナビゲーション**: repo スコープ。`Repositories` で repo に対し `b`=Branches、`s`=Source(既定ブランチ root)。既存: `Enter`=PullRequests、`p`=Pipelines。`PullRequests` からも `b`/`s` で行けると良い。各画面 `Esc`=戻る。
- **Branches(一覧)**: 各行 = `ブランチ名  最終コミット(短縮hash)  日時  概要1行`。`Enter`=そのブランチの Commits、`s`=そのブランチの Source root、`r`=再読込。ページング対応。
- **Commits(履歴)**: 各行 = `短縮hash  日時  author  メッセージ1行目`。`Enter`=CommitDetail、`r`=再読込。ページング。
- **CommitDetail**: メタ(hash/author/date/parents)+ メッセージ全文。`d`=当該コミットの Diff(M1 ビューア)、`Esc`=Commits。
- **Source(ツリー閲覧)**: 現在の `ref` と `path` をヘッダ表示。エントリ一覧(ディレクトリ→ファイルの順、ディレクトリに `/` 印)。`Enter`=ディレクトリなら潜る/ファイルなら FileView、`Backspace`/`Esc`=親ディレクトリ(root で repo へ戻る)、`r`=再読込。ページング。
- **FileView**: ファイル内容をページャ表示(`↑↓ j/k`・`PgUp/PgDn`・`g/G`)。ヘッダに path。バイナリ/巨大は「(バイナリ表示不可)」/先頭 N 行+打切り。`Esc`=Source。
- 共通キー踏襲(`q`/`Ctrl+C`/`Esc`/`?`)。ヘルプに M3 キー追記。非同期は M0〜M2 同様 `tokio::spawn`→`Msg`。ローディング/エラーはステータス行。

## モジュール変更(目安)

- `api/models.rs`: 上記モデル追加。`api/client.rs`: `list_branches`/`list_commits`/`get_commit`/`get_commit_diff`(text)/`list_src`(dir)/`get_src_file`(text)。ディレクトリ/ファイルの出し分けは `type` または別メソッドで。
- `tui/app.rs`: `Screen`/`Msg`/`Command` 拡張。`tui/ui.rs`: 新画面描画。`tui/event.rs`: 新 Command dispatch。`Source` のパス操作(潜る/戻る)状態を持つ。FileView は `logview`(M2)を再利用。commit 差分は M1 の diff モジュールを再利用。
- `docs/design.md` の画面遷移を更新。

## 品質バー(M0〜M2 と同じ、全て green)

- `export PATH="$HOME/.rustup/toolchains/stable-aarch64-apple-darwin/bin:$PATH"` の上で
  `cargo fmt --check` / `cargo clippy --all-targets --offline -- -D warnings` / `cargo test --offline`。
- `unwrap`/`expect` はテスト以外禁止。bin=anyhow, api=thiserror。
- ネットを叩くテストは `#[ignore]`。通常テストはモック: Branch/Commit/SrcEntry デシリアライズ、dir/file 判定、path 潜り/戻りロジック、バイナリ判定、`update()` の遷移。commit diff は M1 の diff テストで担保。
- TUI 中 stdout/stderr 禁止。端末は既存 `Tui` 再利用。crossterm 型は `ratatui::crossterm` 経由。ローカル絶対パス(`/Users/...`)は使わず `$HOME`/`~`(leak guard 回避)。

## 受け入れ条件(Acceptance)

- [ ] fmt/clippy(-D warnings)/test すべて green(offline)。
- [ ] repo から Branches 一覧に入れる。
- [ ] ブランチ選択 → Commits 履歴表示。
- [ ] コミット選択 → CommitDetail、そこから Diff を色付き表示(M1 流用)。
- [ ] repo から Source root に入れ、ディレクトリを潜る/親へ戻る操作ができる。
- [ ] ファイル選択 → 内容をページャ表示。バイナリ/巨大は安全に代替表示。
- [ ] 404/エラーがステータス行に表示。

## 検証方法

- ユニット: 上記モック検証。ネット依存は `#[ignore]`。
- 手動(実 token 必要・任意): ブランチ→履歴→コミット差分、Source でディレクトリ潜行→ファイル表示を一巡。手順を README に追記。
- 実 API で判明したフィールド/挙動(src の dir/file 応答形・diff spec・commit フィールド)を `docs/LEDGER.md` に反映。
