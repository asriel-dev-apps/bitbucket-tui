# M7: コメントUIの磨き込みと PR フィルタ

実運用フィードバック第 3 陣。Phase 1（コメントUI磨き + キーヒント）と Phase 2（PR フィルタ）の 2 段で実装する。

## Phase 1

### 1.1 スレッド間の間隔

**問題**: あるスレッドのアクション行/枠下端と、次のスレッドの枠上端（または次の diff 行）が密着していて読みにくい。

**仕様**:
- `CommentRowKind::Spacer` を追加し、**各スレッドブロックの直後**（`push_thread_rows` の `Bottom` の後、`push_collapsed_row` の `Collapsed` の後）に 1 行挟む。
- `Spacer` は non-focusable（`is_focusable` は Header/Collapsed のみ true の現行 match で自動的に false になることを確認）。
- 描画（`comment_box_line`）: 空行（unified/split 両方。split は左カラムも空行）。
- `click_body_row`: `Spacer` クリックは **no-op**（現在の「comment_id 無し → スレッド先頭選択」分岐に落ちないよう、明示的に無視する）。
- ヒットボックス収集（Actions）には影響しない。

### 1.2 コメントエディタのカーソル表示・移動

**問題**: 返信/編集ポップアップで文字入力位置が見えない。

**仕様**:
- `CommentEditor` に `cursor: usize`（`text` の**char 単位**の挿入位置）を追加。`inline`/`reply` は 0（空文字）、`edit` はプリフィル本文の**末尾**で初期化。
- `on_key_comment_editor`:
  - 文字入力/改行（Enter）はカーソル位置に**挿入**、Backspace はカーソル直前を削除（いずれも char 境界で安全に。byte index 演算をしない）。
  - `Left`/`Right`（1 文字）、`Home`/`End`（**論理行**の行頭/行末）、`Up`/`Down`（前/次の論理行へ。列は char 数で維持し行長にクランプ）を追加。
  - 既存の `Ctrl+S` 送信 / `Esc` 取消は不変。
- `render_comment_editor`: カーソル位置の 1 文字を `Style::new().reversed()` で反転表示。カーソルが行末（その行の文字数と同位置）のときは反転させた空白 1 個をその行末に追加（Onboarding の `input_spans`(ui.rs) と同じ視覚方式）。折返し（`Wrap`）は span 単位で維持されるため、反転 span をカーソル char に付けるだけでよい（自前の折返し計算はしない）。
- プレースホルダ（空文字時の「（コメントを入力）」）は反転空白カーソルを先頭に出したうえで維持してよい。

### 1.3 選択ハイライトの可読性

**問題**: 全テーマで `selection_bg == accent == info` のため、選択中コメントの bg 上書きで marker/author/リンク（accent/info の fg）が背景に溶けて読めない。

**仕様**:
- `comment_box_content_line` の hot 時、`highlighted_diff_line` と同じ方式で **全 span の style を `bg(selection_bg) + fg(selection_fg)` に上書き**する（span 個別 fg が Line style より優先されるため、span 単位の上書きが必須）。枠線 span も含めてよい（diff 行ハイライトと同じ見た目に揃える）。
- `Collapsed` の hot 分岐も同様に fg/bg 両上書きへ変更。
- テーマ定義（theme.rs）は**変更しない**。

### 1.4 キーヒントのはみ出し

**問題**: 端末幅が狭いと、フッターのキーヒントが右端で見苦しく切れる。

**仕様**:
- `render_hints` のループで累積幅を追い、**次のエントリが丸ごと収まらなくなったら break**（部分描画しない。省略記号も付けない）。
- ヒットボックス push も同じ break に従う（描画されないエントリはクリック不可のまま）。
- `hint_entries` のデータは変更しない。

### Phase 1 品質ゲート・不変条件

- `cargo fmt --check` / `cargo clippy --all-targets -- -D warnings` / `cargo test --offline` 全 green（現在 543 pass / 1 ignored）。
- 新規依存の追加禁止。テスト外の `unwrap()`/`expect()` 禁止。色は必ず `theme` 経由（状態層 app.rs に `Color::` を持ち込まない）。
- 壊してはならない既存動作: 表示行モデル（`display_rows` の添字 = scroll/cursor）、`rendered_lines`/`rendered_split` の描画キャッシュ、Actions 行のヒットボックス座標、コラプスのトグル/自動コラプス、split の左右対応。
- 新規テスト（最低限）: Spacer が各スレッド後に入り non-focusable であること（display_rows 構成）、エディタのカーソル挿入/移動（Left/Right/Home/End/Up/Down・途中挿入・途中 Backspace）、hot 行の全 span が selection_fg になること、render_hints が狭い幅でエントリ境界で止まること。

## Phase 2（PR フィルタ。Phase 1 完了後に着手）

**ゴール**: PR 一覧で Web と同様に「state の複数選択（例: OPEN+MERGED）」と「author 指定」でサーバサイドにフィルタできる。

### API（事前調査済み。公開リポジトリで実測確認）

- `state` パラメータの繰り返し（`?state=OPEN&state=MERGED`）は OR として有効。値は OPEN/MERGED/DECLINED/SUPERSEDED。
- `q=`（BBQL）: `state IN ("OPEN","MERGED")`・`author.uuid="{uuid}"`・AND 連結・`sort` 併用すべて有効。
- **重要（実測）: `q` と `state` パラメータを同時指定すると `state` 側が黙って無視される**。混在させないこと。
- `q` の値は全体を URL エンコードする（reqwest のクエリ機構に委ねる）。
- クエリ構築規則: **author 指定なし → 従来どおり `state` 繰り返し**（実績のある経路を維持）。**author 指定あり → `q=state IN (...) AND author.uuid="{uuid}"` に一本化し `state` パラメータは付けない**。いずれも `sort` は併用。

### フィルタ状態

- `PrStateFilter`（enum）を「states の集合 + author」のフィルタ構造体へ拡張（例: `states: BTreeSet<PrState>`、`author: Option<PrAuthor{uuid, display_name}>`。Hash/Eq 可能に）。
- 既存の `o`/`m`/`d`/`a` 単発キーは「その state のみ（All は 4 つ全部）」の集合へ写像して維持。
- `PullRequestsCache` のキーへ author（uuid）と states 集合を含める（キー衝突による誤表示を防ぐ）。
- 永続化: **states の選択は config.toml に保存**（文字列配列）。author はセッション内のみ（リポジトリ依存のため保存しない）。

### UI

- PR 一覧で `f` を押すとフィルタモーダル（`f`/`F` は未割当を確認済み。`a` は既存 All で使用中のため使わない）。
- モーダル構成: 上段 **State**（4 つのチェックボックス。`↑↓/jk` 移動・`Space` トグル）、下段 **Author**（`All authors` + 候補リスト。`↑↓/jk` 選択）。`Tab` でセクション移動、`Enter` で適用（1 ページ目から再取得）、`Esc` で取消。states が 0 個で Enter したら Status エラー（適用しない）。
- author 候補: **ワークスペースメンバー API**（`GET /2.0/workspaces/{workspace}/members`、応答は `values[].user`。ページング追従）から**モーダル初回表示時に遅延取得しワークスペース単位でキャッシュ**。取得失敗（権限等）時は**読み込み済み PR の author から重複排除して**候補にする（フォールバック。uuid 無しの author は候補から除外）。
- フィルタ適用中はタイトル等にフィルタ内容を表示（例: ` PR (OPEN+MERGED, author: Alice) `）。

### Phase 2 品質ゲート・不変条件

- Phase 1 と同じゲート・不変条件。
- 追加: members API の応答形（`values[].user` の uuid/display_name）・`q` の実挙動は「未検証の仮定」として LEDGER に記録する（読み取り系なので実 token での検証はユーザーに委ねる）。
- 新規テスト: クエリ構築（author 有無での `q`/`state` 切替、URL エンコードは reqwest 委譲の確認）、フィルタ構造体のキャッシュキー、モーダルのキー操作（トグル・適用・空集合拒否）、単発キー写像、config 永続化 round-trip。
